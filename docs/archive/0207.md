# 1

这个项目的语音方案有两个：
1，默认使用 TTS 本地语言引擎，拿圣经卷章节文本生成
2，人声语音包是分级的，有两个等级，当有 wifi 的时候在线更新到本地，更新后再使用人声语音包。

这种方案，是为了更大程序降低初次安装的 size。

默认情况下，为减少安装包的大小，资源只内嵌小语言引擎、简体/繁体圣经文本。

"Ting-Ting" 是什么？
它不是内嵌在 APP 里的。它是 macOS/iOS 系统自带的“御用”中文语音包。
优点：在苹果设备上效果极好，不用下载，不占体积。
缺点：出了苹果生态（比如 Android、Windows），它就不存在了。

安卓/Win 上会变成什么样？
安卓会调用 Google 语音或手机厂商（如小米/华为）自带的语音，效果千差万别，大概率不如 Ting-Ting 自然。
这就是为什么通过 TTS 无法保证“所有平台效果相同”。

语音包也可以下载，默认只有简体中文圣经文本包，可以在有 wifi的情况下，下载繁体文本包。

关于机械朗读、真人朗读，不要放在这里，放在设置里。这里只显示当前是什么朗读方式。

写一个 server，用 python 吧，在本地模拟提供语言包、语音包的下载：一个繁体语言包，两个语音包。都要有压缩。下载后，程序要在本地解压。

程序在设置里，要有下载语言包、语音包的选项，点击后，调用 server 下载。下载后，在设置里就可以选择使用了。
